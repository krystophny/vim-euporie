*euporie.txt*  Interactive Jupyter cells in Euporie beside classic Vim

INTRODUCTION                                            *vim-euporie*

vim-euporie opens and manages an Euporie console in a tmux pane. Code is
edited in Vim and executed in a project-local Jupyter kernel. Rich output is
rendered by Euporie using the outer terminal's graphics protocol.

The default target is Vim 9 inside tmux 3.4 or newer inside a Sixel-capable
terminal. Vim does not need to be compiled with Python support. The external
Python environment is resolved by uv. tmux manages Sixel output natively so
figures scroll, crop, and redraw with the pane.

COMMANDS                                                *euporie-commands*

:EuporieStart       Start or attach to the project sidecar.
:EuporieStop        Stop the project sidecar, kernel, and pane.
:EuporieRestart     Restart them.
:EuporieFocus       Focus the managed tmux pane.
:EuporieStatus      Show pane, kernel, and client status.
:EuporieDoctor      Check each hop of the terminal, tmux, and Vim chain that
                    Shift-Enter and graphics depend on, and report what is
                    wrong. Run it from a Python buffer.
:EuporieInterrupt   Interrupt the kernel.
:EuporieSendCell    Execute the current # %% cell.
:EuporieRunCell     Execute the cell and move to the next one.
:[range]EuporieSend Execute a line range.
:EuporieSendLine    Execute the current line.
:EuporieSendFile    Execute the entire buffer.

PLUG MAPPINGS                                           *euporie-mappings*

vim-euporie defines these buffer-local mappings by default:
>
  <NL>                   Run cell and advance (send Visual selection); this
                         is Ctrl-J, which every terminal can report
  <M-CR>                 Same
  <S-CR>                 Same, when the outer terminal can report it
  nmap <localleader>er <Plug>(EuporieRunCell)
  nmap <localleader>ec <Plug>(EuporieSendCell)
  nmap <localleader>el <Plug>(EuporieSendLine)
  xmap <localleader>e  <Plug>(EuporieSendVisual)
  nmap <localleader>ei <Plug>(EuporieInterrupt)
  nmap <localleader>ef <Plug>(EuporieFocus)
<
The <Plug> mappings are installed for Python buffers.
Alt-Enter and Shift-Enter run the current cell and advance in Normal and
Insert mode. In Visual mode they send the selection. Existing mappings are
kept. Shift-Enter requires an outer terminal that reports it distinctly
(kitty, Ghostty, WezTerm, foot, xterm). VTE-based terminals such as XFCE
Terminal send Shift-Enter as a plain Enter, so use Alt-Enter there.

CONFIGURATION                                           *euporie-config*

g:vim_euporie_auto_start        Default: 1
g:vim_euporie_pane_percent      Default: 40
g:vim_euporie_split             "horizontal" or "vertical"
g:vim_euporie_graphics          "auto", "sixel", "kitty-unicode", or
                                "kitty"; default: "auto". "auto" probes
                                whether tmux itself can render Sixel and
                                falls back to "kitty-unicode" if it cannot.
g:vim_euporie_configure_tmux    Enable allow-passthrough for non-Sixel
                                graphics modes; default: 1
g:vim_euporie_configure_keyboard Enable tmux/Vim modified keys; default: 1
g:vim_euporie_full_screen       Keep executed cells in Euporie's live layout,
                                which is what makes ipywidgets usable, and
                                enable mouse support; default: 1
g:vim_euporie_idle_timeout      Seconds after final Vim detaches; default: 0
g:vim_euporie_start_timeout     First-start timeout; default: 90
g:vim_euporie_with              uv overlay packages; default:
                                ["euporie", "ipykernel", "ipympl"]
                                ipympl provides `%matplotlib widget`, which
                                gives a live canvas: 3D axes can be dragged to
                                rotate and the usual matplotlib toolbar works.
                                Drop it from the list to avoid installing
                                matplotlib.
g:vim_euporie_uv_args           Extra uv arguments; default: []
g:vim_euporie_euporie_args      Extra Euporie arguments; default: []
g:vim_euporie_no_mappings       Only define <Plug> mappings; default: 0

Set b:vim_euporie_root or g:vim_euporie_root to override project detection.
Otherwise the nearest pyproject.toml, uv.lock, or .git directory is used.

Each tmux window and project root shares one pane and kernel. Clients send
heartbeats, so the pane survives a handoff between Vim instances and is
removed automatically after the final client exits. Set a positive
g:vim_euporie_idle_timeout to keep it alive briefly during a handoff.

Python files containing a PEP 723 "# /// script" metadata block use the
dependencies declared in that block and receive a separate pane and kernel.
Create or update such metadata with:
>
  uv add --script analysis.py matplotlib
<
Save the script before starting, and use :EuporieRestart after changing its
dependency metadata.
